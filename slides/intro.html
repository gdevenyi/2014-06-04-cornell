Hello, and welcome to Software Carpentry.

Our Mission

Teach basic lab skills for scientific computing that will make you more productive.

Our goal is to teach basic lab skills for scientific computing to researchers in fields as diverse as cosmology, oncology, and economics.

We will cover the basics of the Unix shell, version control with Git, programming in Python, and using SQL. What we're really teaching, though, is how to automate repetitive tasks, how to track and share work, how to grow a program in a modular, testable, reusable way, and how to structure data.

Seven Plus or Minus Two

  • Short-term memory can only hold 7±2 items at once.
  • Must structure programs and programming to fit this hardware limit.
Three themes will run through our teaching. The first is that human short-term memory can only keep about seven things straight at once. If the number of facts you need to hold in your head to write or understand the next line of code crosses this threshold, your error rate will go up and your productivity will go down. The most important thing in programming is therefore to structure software so that you only need to pay attention to a few things at once.

Mental Fatigue

  • Brains get tired after 45-90 minutes of concentration.
  • Must structure work so that each task can be completed in this time or less.
  • And move around a bit between sprints.
The second theme is that your brain gets tired just like any other part of your body. More specifically, under most circumstances you can only concentrate for 45-90 minutes before you stop absorbing new information. The second most important thing in programming is therefore to break down programming (and other things) so that every task can be completed in this much time or less, and to get up and move around for a few minutes between each work sprint in order to flush fatigue poisons from your brain.

Mental Fatigue (cont.)

  • Five 8-hour days per week maximizes total productivity (not just productivity per hour).
    • See Evan Robinson's Why Crunch Mode Doesn't Work for a summary.
  • So get some rest.
A corollary is that a century of research on sleep deprivation has shown that forty hours of work per week, spaced out, maximizes total productivity. Any more than this, and the errors you're making don't just cancel out the extra time: they actually set you back. If you really want to get things done, you should take a walk, read a book, cook a meal, and most importantly, get some sleep.

Science!

  • We actually know a lot about both learning and software development.
    • How Learning Works
    • Making Software: What Really Works and Why We Believe It
  • Base what and how we teach on the best available evidence.
The third and final theme of this class is that we actually know a fair bit about learning and software development—not everything, but more than most people realize. As far as possible, we base what we teach and how we teach it on the evidence summarized in these two books, because that's what scientists are supposed to do.
